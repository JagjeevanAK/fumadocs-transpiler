### FAQ Accordion

<Accordions type="single">
  <Accordion title="What is Fumadocs Transpiler?">
    A powerful tool that converts annotated Markdown files to Fumadocs-compatible React components with bidirectional conversion support.
  </Accordion>
  <Accordion title="How do I install it?">
    Use npm install -g fumadocs-transpiler to install it globally, or npm install fumadocs-transpiler for local installation.
  </Accordion>
  <Accordion title="Can I reverse the transpilation?">
    Yes! Use the --reverse flag to convert MDX files back to annotated Markdown format.
  </Accordion>
  <Accordion title="Is it open source?">
    Yes, it's available on GitHub under the MIT license with full source code access.
  </Accordion>

## Code Examples

### JavaScript Example

<CodeBlock lang="javascript" title="Basic Usage">
import { FumadocsTranspiler } from 'fumadocs-transpiler';

const transpiler = await FumadocsTranspiler.create();
await transpiler.processFiles({
  input: './docs',
  output: './src/pages',
  description: 'API documentation'
});
</CodeBlock>

### Bash Commands

<CodeBlock lang="bash" title="CLI Commands">
# Forward transpilation
fumadocs-transpiler --input ./docs --output ./src/pages

# Reverse transpilation
fumadocs-transpiler --input ./src/pages --output ./docs --reverse
</CodeBlock>

## Regular Markdown Content

All regular Markdown syntax is preserved during reverse transpilation:

- **Bold text** remains bold
- _Italic text_ remains italic
- `Inline code` stays as inline code
- [Links](https://example.com) are preserved
